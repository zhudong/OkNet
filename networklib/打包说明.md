# 网络请求模块打包说明

## 概述

`networklib` 模块已经配置好，可以打包成JAR或AAR文件供其他项目使用。

## 快速开始

### 生成JAR包

```bash
# 在项目根目录执行
gradlew :networklib:jarRelease
```

生成的JAR文件位置：`networklib/build/libs/networklib-1.0.0-release.jar`

### 生成AAR包（推荐）

```bash
# 在项目根目录执行
gradlew :networklib:assembleRelease
```

生成的AAR文件位置：`networklib/build/outputs/aar/networklib-release.aar`

## 详细说明

### JAR vs AAR

- **JAR**：纯Java类文件，不包含Android资源
- **AAR**：Android资源包，可以包含资源文件、AndroidManifest等（推荐用于Android项目）

### 使用JAR的注意事项

1. JAR文件只包含编译后的类文件，不包含依赖库
2. 使用时需要单独添加以下依赖：
   - `com.lzy.net:okgo:3.0.4`
   - `com.lzy.net:okserver:2.0.5`
   - `com.google.code.gson:gson:2.8.1`
   - `androidx.appcompat:appcompat:1.6.1`

### 使用AAR的优势

1. 可以包含Android资源文件
2. 可以包含AndroidManifest.xml
3. 更适合Android Library项目
4. 依赖管理更清晰

## 模块内容

打包后的JAR/AAR包含以下类：

```
com.fyb.networklib
├── api
│   ├── NetworkApi          # 核心网络请求API
│   └── TripManageApi       # 行程管理API示例
├── data
│   └── BaseEntity          # 基础响应实体
└── util
    ├── JsonCallback        # JSON回调基类
    ├── JsonConvert         # JSON转换器
    ├── TokenProvider       # Token提供者接口
    ├── Convert             # JSON转换工具
    ├── LzyResponse         # 响应包装类
    └── SimpleResponse      # 简单响应类
```

## 验证打包结果

打包完成后，可以检查生成的文件：

1. **JAR文件**：`networklib/build/libs/networklib-1.0.0-release.jar`
2. **AAR文件**：`networklib/build/outputs/aar/networklib-release.aar`

可以使用解压工具查看JAR/AAR的内容，确认包含所需的类文件。
